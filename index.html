    • Discussion

    A python module , ruleN.py ( direct link: here ) has been written to solve this problem. Among the functions it exposes, the most notable ones are:

    • occupyGrid(grid: np.ndarray, N: int)
      Here grid is a 2D numpy array that represents the cells.
      The integer N represents the N-th rule, $ 0 < n < 255 $
      Returns void
      This function, upon execution, occupies the grid with $0$s or $1$s according to the rule. It skips the boundaries (e.g. first column, last column), since for the cells in the boundary columns, there are only two arguments from the previous row to evaluate.
    • showMatrixPlot(grid: np.ndarray), Returns void
      Shows the occupied grid.
    • saveMatrixPlot(data: grid, N: int, relPath: string) Returns void
      Iterates up to N number of times and generates images of 0 to N Rule, then saves into the given relative path (e.g. './images/' )
    The checking of which cell should be occupied is done in occupyGrid(grid, N) function.
